# Restrição 1: Limites de tamanho da wave (unidades a coletar)
RESTRICAO: Limites_Wave
DESCRICAO: A quantidade total de unidades a serem coletadas deve estar dentro dos limites LB e UB
EXPRESSAO: LB ≤ ∑(o∈O') ∑(i∈I(o)) u(oi) ≤ UB
TIPO: BOUND
IMPLEMENTACAO: DEFAULT

# Restrição 2: Disponibilidade de itens nos corredores
RESTRICAO: Disponibilidade_Itens
DESCRICAO: Os corredores selecionados devem ter estoque suficiente para todos os itens dos pedidos
EXPRESSAO: ∑(o∈O') u(oi) ≤ ∑(a∈A') u(ai), ∀i∈I(o) com o∈O'
TIPO: CAPACITY
IMPLEMENTACAO: DEFAULT

# Restrição 3: Pedidos indivisíveis (implícita na modelagem)
RESTRICAO: Pedidos_Indivisiveis
DESCRICAO: Um pedido deve ser atendido completamente ou não ser atendido
EXPRESSAO: Implícita na seleção de pedidos completos
TIPO: STRUCTURAL
IMPLEMENTACAO: DEFAULT
